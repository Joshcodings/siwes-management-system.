System Implementation

Component Implementation Details
Frontend Interface Olive/Gold-themed, glassmorphism UI built with React 19 + Vite + TailwindCSS 4. Framer Motion (motion/react) powers all page transitions and micro-animations. Persistent sidebar navigation with Dashboard, Placement, Logbook, Applications, Memos, Documents, and Profile tabs — with full mobile responsiveness via a collapsible hamburger menu.
Backend Architecture Express.js (Node.js) serves a modular REST API via server.ts. Helmet.js enforces HTTP security headers; express-rate-limit guards all endpoints. better-sqlite3 executes synchronous, atomic database transactions — preventing race conditions on concurrent logbook submissions or application approvals.
Database Layer SQLite database (siwes.db) managed via better-sqlite3. Schema supports six core tables: users, student_profiles, companies, applications, logbook_entries, and memos. Migration scripts (migrate.cjs) handle schema evolution. RBAC permissions are enforced at every query level by role (SUPER_ADMIN, ADMIN, SCHOOL_SUPERVISOR, STUDENT).
User Authentication JWT tokens issued on login; stored in React state and attached to every API request via the Authorization header. Passwords hashed with bcryptjs. Custom middleware (authenticate + authorize) on every protected route extracts the userId and role from the JWT payload, isolating multi-tenant data with strict RBAC enforcement.
AI Placement Engine Python script (ai_engine.py) spawned server-side via Node.js spawnSync. Computes a 100-point composite score per company using: Skill Overlap (0–60 pts), Course Relevance (0–30 pts), and Location Proximity via the Haversine formula (0–10 pts). Results are ranked and returned as JSON to the frontend for display as AI-powered recommendations.
GPS Geofence System Browser Geolocation API streams the student's live coordinates. Server calculates real-time distance from the student's registered internship site using the Haversine formula. A configurable geofence radius (default: 300m, adjustable by Super Admin) gates logbook submission — students outside the perimeter cannot submit, preventing fraudulent off-site logging.
Data Visualization Recharts library renders interactive PieCharts (Placement Progress) and BarCharts (7-Day Daily Logbook Activity) on the Admin dashboard. The Student Logbook tab features a 14-Day Gamification Streak Heatmap inspired by GitHub contribution calendars — showing verified, pending, flagged, and missed days to encourage consistent daily engagement.
Document Generation jsPDF + jspdf-autotable generate all official ITF forms on-demand: SIWES Letter of Introduction, SCAF/SPE-1 (Commencement Form), ITF Form 8 (Monthly/Final Appraisal), and a full Student Logbook PDF. All documents are pre-populated with live student and company data from the database and downloaded directly in the browser without any server roundtrip.
